这组文章来自我跟着 FastAPI 官方教程做的入门笔记。官方教程本身写得很清楚，但它的组织方式带有很强的“查询手册”味道：想到一个特性，就切成一个独立小节。这对查文档很友好，但对连续学习来说会显得有些碎。

官方教程页自己也明确写了，这些章节一方面是循序渐进的，另一方面又围绕单独主题组织，方便你“直接跳到某个章节以解决特定 API 需求”。这正好解释了为什么它拿来系统学习时，会有一种“刚摸到一点整体，又被切回单点功能”的感觉。官方入口在这里：<https://fastapi.tiangolo.com/zh/tutorial/>

基于我现在已经写到的 1-12 份笔记，我把它重新收成了下面这条学习线：

1. `起步：第一个应用与并发直觉`
先用最小例子跑通 FastAPI，然后用“做汉堡”的异步示例建立并发心智。

2. `路径参数与查询参数`
把 URL 里的两类核心输入分清楚：哪些是路径的一部分，哪些是查询条件，顺手理解自动文档和类型转换。

3. `请求体与 Pydantic 模型`
从单个请求体开始，到多个请求体参数、嵌套模型、示例数据，把“客户端提交的数据”这一层完整捋顺。

4. `参数校验与请求来源`
把 `Query / Path / Body / Cookie / Header` 统一成一个心智模型：参数从哪里来，又怎么做约束和校验。

5. `响应模型与常见数据类型`
最后回到输出端，理解响应模型、返回类型以及 Pydantic 支持的常见类型系统。

这样整理之后，学习顺序就不再是“一个特性一个特性点着学”，而是：

- 先知道 FastAPI 的请求流长什么样
- 再理解输入参数和请求体
- 然后进入更细的校验
- 最后再回到输出与类型

这组文章目前只覆盖我已经写下来的 1-12 部分，所以它还不是完整的 FastAPI 路线图。后面如果我继续写到状态码、表单、文件上传、错误处理、依赖注入、安全、数据库和部署，这条路线会继续往后长。

正文里我尽量保留了原始代码和原本的理解，只做了这些整理：

- 把原先过碎的章节重新分组
- 补了更适合阅读的导语和小标题
- 尽量用“请求从哪里来、数据往哪里去”的角度来串内容

如果你也是第一次系统学 FastAPI，建议就按这里的顺序往下读。
